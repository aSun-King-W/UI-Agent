# UI自动化测试Agent Skill开发任务要求

## 一、任务背景

本题目旨在考察候选人在 Agent 开发领域的综合能力，重点评估以下维度：

- Agent 架构设计能力
- Skill 开发规范理解
- 自动化测试场景建模
- 问题分解与技术选型

> **重要说明**：虽然题目涉及 UI 自动化测试专业知识，但在 AI 时代，Agent 开发工程师的核心能力在于将复杂任务抽象为可执行的 Skill，而非精通所有领域细节。我们鼓励借助 AI 工具完成设计。

---

## 二、业务需求描述

### 2.1 业务场景

设计并实现一个 UI 自动化测试 Skill，完整实现以下端到端流程：

1. **任务下发**：通过飞书接收测试任务指令
2. **浏览器自动化**：访问淘宝网站 (`https://www.taobao.com/`)
3. **用户登录**：完成淘宝账号登录
4. **商品搜索**：搜索关键词 `"索尼耳机"`
5. **智能筛选**：找到好评率 ≥ 99% 的商品
6. **购物车操作**：将符合条件的商品加入购物车
7. **结果反馈**：将测试结果回传飞书

### 2.2 技术约束

- **框架兼容**：Skill 需符合 OpenClaw 框架规范
- **编排方式**：禁止硬编码线性工作流编排（需具备动态决策与自适应能力，例如根据页面状态、中间结果、异常情况动态调整下一步操作）
- **通用性**：设计需要具备一定的通用性和泛化能力，可复用至类似场景（如不同电商平台、不同搜索关键词、不同筛选条件）
- **凭证安全**：敏感信息（淘宝账号密码、飞书 Token 等）需通过安全机制管理，禁止硬编码在代码中

### 2.3 环境要求

- **运行环境**：Python 3.10+ 或 Node.js 18+
- **浏览器驱动**：需兼容无头/有头浏览器模式（如 Playwright / Selenium）
- **依赖管理**：所有依赖需通过 `requirements.txt` 或 `package.json` 声明

## 三、交付要求

候选人需以技术设计文档及代码形式提交方案，包含但不限于以下内容：

### 3.1 架构设计

- Skill 整体架构图（可用 Mermaid 或其他图形化方式呈现）
- 核心模块划分及职责说明
- 技术栈选型及理由（例如浏览器自动化工具、飞书 SDK、登录方案等）

### 3.2 Skill 定义

- `SKILL.md` 文件结构设计（需给出完整或关键部分的定义示例）
- 相关脚本设计（如脚本组织方式、参数传递、错误处理策略）

### 3.3 关键技术点分析

需识别并阐述实现过程中可能存在的关键技术难点及解决思路，涵盖但不限于以下方面：

- **淘宝登录反自动化检测**：如滑块验证码、设备指纹、风控策略等
- **动态页面元素定位**：如异步加载、类名变化、弹窗遮挡等
- **好评率筛选逻辑**：如何在页面结构中提取好评率数据并做条件过滤
- **飞书消息交互**：机器人消息接收解析、富文本结果回传
- **非硬编码走编排**：Agent 如何感知当前页面状态，动态决策"做什么、怎么做、下一步做什么"

### 3.4 代码示例

提供核心代码片段，能体现以下能力：

- **Skill 注册与飞书指令接收**：Skill 的入口定义、飞书消息 Webhook 处理
- **浏览器自动化操作**：搜索、筛选、加购等原子操作封装
- **非硬编码的动态决策逻辑**：体现 Agent/LLM 驱动的自适应工作流，而非预定义 if-else 流程。例如：根据页面实际内容判断是直接点击还是需要展开筛选面板，根据筛选结果决定是继续翻页还是终止搜索
- **凭证安全读取**：从环境变量或密钥管理服务获取敏感信息
- **结果组装与飞书回传**：将结构化测试结果以飞书消息卡片或文本形式回传

### 3.5 效果评估报告

- 提供 Agent 效果评估报告，至少包含：
  - 任务成功率（在正常网络与账号条件下的预期表现）
  - 异常场景处理能力（如无符合条件商品、登录验证码拦截、页面结构变更等）
  - 执行效率与资源消耗的简要分析
