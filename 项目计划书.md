> 版本：v1.0
> 周期：2026/05/11 ~ 2026/05/13（2天）
> 目标：交付完整可运行的 Playwright + Python OpenClaw Skill 原型

---

## 一、项目概述

### 1.1 目标

设计并实现一个基于 OpenClaw 规范的 UI 自动化测试 Skill，完整覆盖以下端到端流程：

飞书接收指令 → 浏览器访问淘宝 → 账号登录 → 搜索"索尼耳机" → 筛选好评率≥99% → 加入购物车 → 结果回传飞书

### 1.2 核心要求

| 要求 | 说明 |
|------|------|
| 非硬编码编排 | 基于 LLM/Agent 动态决策，非预定义 if-else 流程 |
| 可泛化 | 可复用到不同电商平台、关键词、筛选条件 |
| 凭证安全 | 敏感信息通过环境变量管理，禁止硬编码 |
| 反自动化对抗 | 处理淘宝登录反检测机制 |

### 1.3 技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| 浏览器自动化 | Playwright | 原生 auto-wait、多浏览器支持、Stealth 生态好 |
| 运行时 | Python 3.10+ | OpenClaw SDK 支持、生态成熟 |
| 编排驱动 | LLM (OpenAI/Claude API) | 实现非硬编码动态决策 |
| 飞书交互 | 飞书开放 API 自建机器人 | 支持消息接收与富文本回传 |
| 反检测 | playwright-stealth | 绕过淘宝风控检测 |
| 凭证管理 | python-dotenv + .env | 敏感信息隔离 |

### 阶段二：原子操作层（Day 1 下午 · ~5h）

> 目标：完成浏览器自动化的核心原子操作，每个模块可独立测试

#### 2.1 浏览器引擎 — `core/browser.py`

```
功能：
- 启动 Chromium 浏览器（有头/无头模式切换）
- 注入 playwright-stealth 反检测脚本
- 统一的页面上下文管理
- 超时与重试策略
```

#### 2.2 淘宝登录 — `core/login.py`

```
功能：
- 访问淘宝登录页
- 切换为密码登录模式
- 输入账号密码
- 处理滑块验证码（尝试绕过策略）
- Cookie/Session 持久化（下次启动自动复用）
- 登录状态检测
```

核心难点：淘宝反自动化检测。应对策略：
- Stealth 脚本注入（修改 navigator.webdriver、WebGL、字体等指纹）
- 慢速模拟真实操作（随机延迟、鼠标轨迹）
- 失败降级：使用已保存的 Cookie 复用 Session

#### 2.3 商品搜索 — `core/search.py`

```
功能：
- 在首页/搜索框输入关键词
- 执行搜索
- 等待搜索结果加载
- 提取商品列表：标题、价格、链接、评价数、好评率
```

#### 2.4 智能筛选 — `core/filter.py`

```
功能：
- 在搜索结果页定位筛选条件
- 找到好评率筛选入口（可能为下拉/滑块/复选框等形式）
- 适配不同页面结构的筛选交互
- 遍历商品列表，提取好评率数据
- 翻页：当前页无≥99%商品时自动翻页
```

#### 2.5 购物车操作 — `core/cart.py`

```
功能：
- 进入满足条件的商品详情页
- 选择规格（如有弹窗）
- 点击「加入购物车」
- 验证是否加购成功
```

#### 2.6 页面工具 — `core/page_utils.py`

```
功能：
- 全页/局部截图
- 提取页面结构化数据（标题、按钮、列表等）
- 智能等待元素出现
- DOM 状态摘要（供 LLM 决策使用）
```

### 阶段三：动态编排与 LLM 决策（Day 2 上午 · ~4h）

> 目标：实现非硬编码的 Agent 驱动工作流

#### 3.1 Agent 编排引擎 — `orchestrator/agent.py`

设计思路：

```
┌─────────────────┐
│  飞书消息接收    │
│  (skill_entry)   │
└────────┬────────┘
         ▼
┌─────────────────┐
│  Agent 决策循环  │  ← LLM 每次根据当前状态决策下一步
│  (orchestrator)  │
└────────┬────────┘
         ▼
┌─────────────────┐
│  Tool 执行       │  ← 调用原子操作模块
│  (core/*)        │
└────────┬────────┘
         ▼
┌─────────────────┐
│  结果回传飞书    │
│  (feishu/bot)    │
└─────────────────┘
```

核心循环伪代码：

```
while steps < max_steps:
    # 获取当前页面状态
    page_state = extract_page_state(page)
    
    # LLM 决策
    decision = llm.decide(
        system_prompt=prompt,
        tools=available_tools,
        context={
            "page_url": page.url,
            "page_state": page_state,
            "last_action": last_action,
            "last_result": last_result,
            "user_goal": "搜索索尼耳机，筛选好评率≥99%，加入购物车"
        }
    )
    
    # 执行决策
    result = execute_tool(decision.tool, decision.args)
    
    # 检查是否完成
    if is_goal_achieved(result):
        break
    
    steps += 1
```

#### 3.2 Tool 定义 — `orchestrator/tools.py`

| Tool 名称 | 功能 | 参数 |
|-----------|------|------|
| `navigate` | 导航到指定 URL | `url: str` |
| `login_taobao` | 执行淘宝登录 | — |
| `search_product` | 搜索商品 | `keyword: str` |
| `apply_filter` | 应用筛选条件 | `condition: dict` |
| `add_to_cart` | 加入购物车 | `product_index: int` |
| `scroll_page` | 滚动页面 | `direction: str` |
| `get_page_state` | 获取页面摘要 | — |
| `extract_products` | 提取商品列表 | — |
| `finish` | 完成并返回结果 | `result: dict` |

#### 3.3 Agent Prompt 设计 — `orchestrator/prompt.py`

System Prompt 核心内容：

- 角色定义：你是一个 UI 自动化测试 Agent，操作淘宝网页端
- 输入格式：当前 URL、页面摘要、上一步操作与结果
- 输出格式：JSON `{ "tool": "...", "args": {...}, "reasoning": "..." }`
- 约束：不可做任何违规操作，仅执行测试任务
- 边界：最大步数限制、失败重试次数

#### 3.4 飞书 <> Agent 流程串联 — `skill_entry.py`

- 监听飞书指令 → 解析任务参数（关键词、筛选条件等）
- 启动 Agent 编排 → 实时状态上报（飞书群内进度通知）
- 收集最终结果 → 结构化回传飞书

## 四、关键风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 淘宝滑块验证码无法绕过 | 高 | 高 | Stealth 方案 + Cookie 复用降级 |
| 飞书应用审核耗时 | 中 | 中 | 先走 Webhook 模拟，不影响开发进度 |
| 页面结构变更导致定位失效 | 中 | 中 | LLM 自适应定位 + 多选择器 fallback |
| 好评率数据格式不统一 | 中 | 中 | LLM 提取而非硬编码 CSS 选择器 |
| 2天时间不够 | 中 | 中 | 优先保证核心流程，文档后补 |
